# Competitive Analysis

## Overview

EefEats is building toward "Goodreads for recipes." Understanding the competitive landscape prevents building features that commodity apps already do well — and surfaces where to differentiate.

## EefEats Competitive Space

| Category | Players |
|----------|---------|
| Recipe saving / bookmarking | Paprika, Yummly, Whisk, AnyList |
| Social recipe sharing | Allrecipes, Food Network, TikTok (creator recipes) |
| Social reading lists (model) | Goodreads, Letterboxd, StoryGraph |
| Creator monetization | Patreon, Substack (newsletter recipes) |
| Cooking tools | Whisk (smart lists), Mela, Crouton |

## Analysis Framework

For any feature or positioning question, evaluate across these dimensions:

### 1. Feature Parity Check
Does this feature exist in Paprika / Yummly / Whisk / Mela?
- If yes: what's our differentiation angle?
- If no: is there a reason competitors skipped it? (cost, complexity, IP risk)

### 2. Social Differentiation
EefEats' core bet: **social context makes recipes better**.
- Does this feature add social signal? (who cooked it, who liked it, who forked it)
- Does it build the social graph? (follows, activity feed, creator profiles)
- Does it enable creator monetization?
If none of the above: is this a commodity feature or a moat-builder?

### 3. Goodreads/Letterboxd Model
Key insight: those apps succeeded because they made personal taste public and social.
- Does this feature help users express taste/preference?
- Does it help users discover through trusted people, not algorithms?
- Does it create a "cooking activity" record (cooked this, rated it, forked it)?

### 4. IP / Legal Angle
- Does this feature risk conflict with recipe copyright holders?
- Does it expose scraped content in a way competitors avoid?
- Could a large player (Allrecipes, NYT Cooking) clone and kill this?

### 5. Monetization Position
- Does this feature fit free tier or premium?
- Does it enable creator subscriptions?
- Does it create brand partnership opportunities?

## Positioning Statement

> EefEats is for people who cook from trusted sources — friends, followed creators, curated originals — not anonymous recipe feeds. Where Allrecipes is Wikipedia, EefEats is Goodreads.

Use this as a filter: if a feature feels more "Wikipedia" than "Goodreads," question whether it belongs.

## Output Format

When completing a competitive analysis, produce:
1. **Competitive matrix**: Feature vs. competitor grid
2. **Differentiation gaps**: Where EefEats is currently behind, at parity, or ahead
3. **Build/skip recommendation**: For each feature evaluated
4. **Risks**: Which competitors could copy this fastest
